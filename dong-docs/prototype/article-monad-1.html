타입스크립트 모나드(1) — Option과 안전한 부재

dongCoding2026.01.128분👁 1,551

null은 사고를 부르는 값이다. 부재를 값이 아니라 타입으로 다루면, 컴파일러가 대신 조심해준다.

Option이라는 상자

값이 있을 수도, 없을 수도 있는 상태를 하나의 타입으로 감싼다.

type Option<T> = { tag: 'some', value: T } | { tag: 'none' }

왜 상자에 넣나

부재를 명시적으로 만들면, 처리하지 않은 none 경로가 타입 에러로 드러난다.
