클로드코드 입문(1) — 설치와 첫 대화

dongCoding2026.06.20수정 2026.07.017분👁 1,284

터미널은 오랫동안 차가운 곳이었다. 그런데 그 안에 대화 상대가 생기면, 도구를 쓰는 감각이 통째로 바뀐다.

이 글에서는 설치부터 첫 대화까지를 다룬다. 인라인 수식은 이렇게 — 복잡도 $O(n \log n)$ — 문장 속에 놓인다. 인라인 코드 claude 도 함께.

설치

패키지 매니저는 pnpm을 쓴다.

# 전역 설치 후 첫 실행
pnpm add -g @anthropic-ai/claude-code
claude "이 저장소를 설명해줘"
도구가 말을 걸어오기 시작하면, '어떻게'보다 '무엇을'에 집중하게 된다.

시킬 것과 시키지 말 것

  • 맡겨라: 반복적 리팩터링, 테스트 작성
  • 직접 하라: 되돌리기 어려운 결정, 배포

$$ \nabla \cdot \mathbf{E} = \frac{\rho}{\varepsilon_0} \qquad e^{i\pi} + 1 = 0 $$

덜 만드는 용기에 대하여

dongCoding2026.05.114분

기능을 더하는 일에는 박수가 따르지만, 빼는 결정에는 침묵만 남는다. 그 침묵을 견디는 게 설계다.

YAGNI(You Aren't Gonna Need It)는 게으름의 변명이 아니다. 아직 증명되지 않은 필요를 미루는 규율이다. 이 글은 시리즈에 속하지 않는 단독 산문이라, 아래에 이전/다음 회차 네비게이션이 붙지 않는다.

복무하지 않으면 넣지 않는다

모든 요소는 하나의 질문을 통과해야 한다 — 읽기를 돕는가, 주의를 훔치는가.

덜어낸 자리에서 비로소 글이 숨을 쉰다.

그래서 무엇을 뺐나

  • 조회수 숫자 — 읽기 실패 시 0이나 에러를 띄우지 않고 그냥 숨긴다.
  • 관련 포스팅 — 매칭되는 글이 없으면 섹션 자체를 그리지 않는다.
